## Script 1: data_collector.py (수정)

### 기존 유지
- PHASE0 마스터 로드
- 기업 분류 (Type A/B/C)
- pykrx: PER, PBR, EPS, BPS, DIV, DPS, 시가총액

### 신규 추가: DART 재무제표 수집
```python
# .env에서 DART_API_KEY 로드
import OpenDartReader
dart = OpenDartReader(os.environ['DART_API_KEY'])

# 수집 대상: 전종목 최근 2개년 사업보고서
# API 호출: dart.finstate_all(code, year, reprt_code='11011', fs_div='CFS')
#   → 연결 재무제표 (일반기업)
# API 호출: dart.finstate_all(code, year, reprt_code='11011', fs_div='OFS')
#   → 별도 재무제표 (지주사용)
```

### DART에서 추출할 항목

**손익계산서 (IS):**
- 매출액
- 영업이익
- 당기순이익
- 이자비용 (이자보상배율 필터용)

**재무상태표 (BS):**
- 자산총계, 부채총계, 자본총계
- 유동자산, 유동부채

**현금흐름표 (CF) — 주주환원 핵심:**
- 배당금의지급 (재무활동)
- 자기주식의취득 (재무활동)
- 자기주식의처분 (재무활동)

### 수집 전략
- 전종목 1,271개 × 2개년 × 연결+별도 = ~5,000건 (일 한도 10,000건 이내)
- 최초 실행: 전체 수집 (2024년, 2025년 사업보고서)
- 이후: 분기 보고서 발표 시 증분 업데이트
- 호출 간격: 0.5초 sleep (서버 부하 방지)
- 결과: cache/dart_cache.pkl

### 출력 데이터 구조
```python
dart_cache = {
    'ticker': {
        'CFS': {  # 연결
            2024: {'매출액': ..., '영업이익': ..., '순이익': ..., ...},
            2025: {'매출액': ..., '영업이익': ..., '순이익': ..., ...},
        },
        'OFS': {  # 별도
            2024: {...},
            2025: {...},
        },
        'CF': {  # 현금흐름 (주주환원)
            2024: {'배당금지급': ..., '자사주취득': ..., '자사주처분': ...},
            2025: {...},
        },
    },
    ...
}
```

## Script 2: consensus_collector.py (신규)

### FnGuide 스크래핑 대상
URL: `https://comp.fnguide.com/SVO2/ASP/SVD_Main.asp?pGB=1&giession=&MenuYn=Y&ReportGB=&NewMenuID=11&stkGb=701&giession=&giession=&giession=&cID=&MenuType=0&Ession=&pagePath=/SVO2/ASP/SVD_Main.asp&domession=comp.fnguide.com&pID=&giession=&giession=&code=005930`

### 수집 항목
- Forward EPS (12개월 선행, 컨센서스 평균)
- Forward 매출액 (컨센서스)
- Forward 영업이익 (컨센서스)
- 목표가 (참고용)
- EPS Revision (3개월 변화율)

### 스크래핑 전략
- PHASE0 마스터에서 시총 상위 ~500종목만 시도
- 컨센서스 데이터 없는 종목 → 자동 스킵
- 호출 간격: 1초 sleep
- User-Agent + 세션 관리
- 결과: cache/consensus_cache.pkl
- Fallback: 실패 시 trailing만 사용 (scoring에서 처리)

### 출력 데이터 구조
```python
consensus_cache = {
    'ticker': {
        'fwd_eps': 5000,          # Forward EPS (12M)
        'fwd_revenue': 300000,     # Forward 매출 (백만원)
        'fwd_op': 50000,           # Forward 영업이익 (백만원)
        'target_price': 80000,     # 목표가
        'eps_revision_3m': 0.05,   # EPS Revision 3개월 (+5%)
    },
    ...
}
```

## Script 3: nav_calculator.py (신규)

### 입력
- data/holdco_list.csv (지주사 목록)
- data/holdco_subsidiaries.csv (지주사-자회사 지분 매핑)
- pykrx 시가총액 (상장 자회사 시총)
- DART 별도 재무제표 (비상장 자회사 장부가, 순차입금)

### NAV 산출 로직
```
NAV = Σ(상장자회사 시총 × 지분율)
    + Σ(비상장자회사 장부가)
    + 자체사업가치 (별도 영업이익 × 8~10x)
    - 순차입금 (별도 기준)

NAV 디스카운트율 = 1 - (시가총액 / NAV)
```

### 출력
- cache/nav_cache.pkl
- 지주사별 NAV 상세 내역

## Script 4: fundamental_scoring.py (전면 수정)

### Phase 0: 데이터 통합
```python
# 캐시 로드
pykrx_data = load('fundamental_cache.pkl')   # pykrx 기본
dart_data = load('dart_cache.pkl')            # DART 재무제표
consensus_data = load('consensus_cache.pkl')  # FnGuide Forward
nav_data = load('nav_cache.pkl')              # 지주사 NAV
```

### Phase 1: 재무건전성 필터
```python
# 필터 조건 (하나라도 해당 → 제외)
filters = {
    '자본잠식': 자기자본 <= 0,
    '고부채': 부채비율 > 400% (금융업 제외),
    '이자보상배율': 영업이익/이자비용 < 1,
    '3년연속적자': 2023~2025 영업이익 모두 < 0,
}
# 필터 적용 후 통과 종목만 스코어링
```

### Phase 2: 지표 산출

**A. Forward 밸류에이션 (30%)**
```python
# Forward PER = 현재가 / Forward EPS
fwd_per = current_price / fwd_eps

# PEG = Forward PER / Forward EPS 성장률
fwd_eps_growth = (fwd_eps / trailing_eps) - 1
peg = fwd_per / (fwd_eps_growth * 100)  # 성장률을 %로

# Fallback: Forward 없으면 Trailing PER (가중치 50% 감소)
```

**B. Forward 성장성 (40%)**
```python
# Forward 매출 성장률 = (Forward매출 / 직전실적매출) - 1
fwd_rev_growth = (fwd_revenue / trailing_revenue) - 1

# Forward 영업이익 성장률
fwd_op_growth = (fwd_op / trailing_op) - 1

# Forward EPS 성장률
fwd_eps_growth = (fwd_eps / trailing_eps) - 1

# EPS Revision (FnGuide에서 직접)
eps_revision = eps_revision_3m

# Fallback: Forward 없으면 Trailing YoY (가중치 50% 감소)
trailing_rev_growth = (rev_2025 / rev_2024) - 1
trailing_op_growth = (op_2025 / op_2024) - 1
trailing_eps_growth = (eps_2025 / eps_2024) - 1
```

**C. 주주환원 (30%)**
```python
# DART CF표에서 추출
total_return = dividend_paid + net_buyback  # 순 자사주매입

# 총 주주환원수익률 = 총주주환원액 / 시가총액
shareholder_yield = total_return / market_cap

# 주주환원/영업이익 = 총주주환원액 / 영업이익
return_to_op = total_return / operating_profit

# 배당성향 = DPS / EPS (또는 배당총액/순이익)
payout_ratio = dps / eps  # 캡: 0~100%

# 자사주매입비율 = 자사주매입금액 / 시가총액
buyback_yield = net_buyback / market_cap
```

### Phase 3: Rank Percentile + 카테고리 스코어링
```python
# 각 지표별 rank percentile (0~1)
# 카테고리 내 평균 → 카테고리 간 가중 합산

score = (valuation_cat * 0.30) + (growth_cat * 0.40) + (shareholder_cat * 0.30)

# Forward/Trailing 가중치 조정
# Forward 있는 종목: 정상 가중치
# Forward 없는 종목: 해당 카테고리 가중치 × 0.5 → 재정규화
```

### Phase 4: 결과 출력
- Excel: 전체랭킹, Top50, 섹터별Top10, 지주사NAV, 필터제외종목
- 차트: Top30 바차트, 밸류에이션vs성장성 산점도, 섹터별 평균, 주주환원 분포

## 구현 순서 (우선순위)

### 1순위: DART 재무제표 + 주주환원 (즉시)
- data_collector.py에 DART 수집 추가
- 현금흐름표에서 배당금/자사주매입 추출
- 재무건전성 필터 구현

### 2순위: FnGuide 컨센서스 스크래핑
- consensus_collector.py 신규 작성
- Forward PER, Forward 성장률 산출
- Forward/Trailing fallback 로직

### 3순위: NAV 계산
- nav_calculator.py 신규 작성
- 지주사 NAV 디스카운트 산출
- 밸류에이션 카테고리에 NAV 지표 편입

### 4순위: fundamental_scoring.py 전면 개편
- 3-카테고리 스코어링 (기존 5-카테고리 대체)
- Forward/Trailing 가중치 조정
- 필터 + 스코어 결합

## 의존성

```
data_collector.py (pykrx + DART)
    ↓
consensus_collector.py (FnGuide) ─── 독립 실행 가능
    ↓
nav_calculator.py (pykrx 시총 + DART 별도)
    ↓
fundamental_scoring.py (모든 캐시 통합)
```

- DART 수집과 FnGuide 스크래핑은 독립적으로 실행 가능
- scoring은 모든 캐시가 준비된 후 실행
- 캐시가 없는 부분은 graceful fallback (해당 카테고리 스킵 또는 가중치 감소)
