## 1. 핵심 이슈: 지주사 더블카운팅 (Double-Counting)

### 문제 정의
한국 시장의 특수성: 재벌 그룹 구조에서 지주사(Holding Company)와 자회사가 동시에 상장
- 지주사의 **연결 재무제표**에 상장 자회사의 실적이 포함됨
- 그런데 상장 자회사는 자체 재무제표로 별도 평가됨
- 결과: 동일 실적이 지주사 + 자회사에서 이중 반영 → 밸류에이션 왜곡

### 대표 사례
| 지주사 | 주요 상장 자회사 | NAV 디스카운트 |
|--------|------------------|----------------|
| SK (034730) | SK하이닉스, SK이노베이션, SKT 등 | ~50% |
| LG (003550) | LG전자, LG화학, LG에너지솔루션 등 | ~50% |
| HD현대 (267250) | HD한국조선해양, HD현대일렉트릭 등 | ~45% |
| 한화 (000880) | 한화에어로스페이스, 한화솔루션 등 | ~55% |
| GS (078930) | GS칼텍스(비상장), GS리테일 등 | ~40% |
| CJ (001040) | CJ제일제당, CJ ENM, CJ대한통운 등 | ~50% |

평균 NAV 디스카운트: ~50% (지주사 시가총액이 순자산가치의 절반 수준)

### 해결 접근법: NAV(순자산가치) 조정

**지주사에 대해서는 연결 재무제표의 PER/PBR/ROE 대신 NAV 기반 밸류에이션 적용**

```
NAV = (상장 자회사 지분가치) + (비상장 자회사 지분가치) + (자체사업 가치) - (순차입금)

상장 자회사 지분가치 = Σ (자회사 시가총액 × 지분율)
비상장 자회사 지분가치 = Σ (자회사 장부가 × 지분율)  [보수적]
자체사업 가치 = 별도 재무제표의 영업이익 × 적정 멀티플
순차입금 = 별도 재무제표의 (차입금 - 현금성자산)
```

## 2. 기업 분류 체계

### Type A: 일반기업 (대다수, ~1,200종목)
- 연결 재무제표 또는 별도 재무제표 기준 일반 펀더멘탈 평가
- 표준 지표 적용 (PER, PBR, ROE, 성장률 등)

### Type B: 지주사 (Holding Company, ~30~50종목)
- NAV 기반 밸류에이션 적용
- 별도(개별) 재무제표 사용 (연결 재무제표는 더블카운팅)
- NAV 디스카운트율 = 1 - (시가총액 / NAV)

### Type C: 금융지주/금융업 (~50~70종목)
- 은행, 보험, 증권: 재무제표 구조가 다름 (이자수익, 보험수익 등)
- PER, PBR, ROE는 적용 가능하나 영업이익률/부채비율은 의미 다름
- 별도 카테고리로 분리하여 섹터 내 상대 비교

### 분류 방법
1. PHASE0 마스터 테이블의 WICS 섹터 활용
   - 금융업 = '은행', '보험', '증권', '기타금융'
2. DART 기업 개황 + 사업보고서에서 '지주회사' 여부 확인
3. 공정거래위원회 지주회사 목록 (연 1회 발표) 활용

## 3. 데이터 소스

### A. pykrx - KRX 기본 밸류에이션 (1순위, 즉시 가능)
- `get_market_fundamental_by_ticker(date)`: 전종목 PER, PBR, EPS, BPS, DIV
- `get_market_cap_by_ticker(date)`: 전종목 시가총액
- 장점: API 키 불필요, 전종목 한번에 조회
- 용도: Trailing PER, PBR, 배당수익률, ROE 근사값(=EPS/BPS), 시가총액

### B. DART OpenAPI (OpenDartReader) - 공식 재무제표 (2순위, API키 필요)
- `dart.finstate(code, year, reprt_code)`: 연결/별도 재무제표
  - reprt_code: '11011'(사업보고서), '11014'(반기), '11012'(1분기), '11013'(3분기)
- `dart.finstate_all(code, year, reprt_code, fs_div)`: 전체 재무제표
  - fs_div: 'CFS'(연결), 'OFS'(별도)
- `dart.report(code, '지분현황')`: 주요 자회사 지분율 → NAV 계산용
- `dart.company(code)`: 기업 개황 (업종, 대표자, 결산월 등)
- 일일 한도: 10,000건 (무료)
- 용도: 영업이익률, 부채비율, FCF, 성장률, **자회사 지분율**, **별도 재무제표**

### C. FnGuide (comp.fnguide.com) - 컨센서스 (3순위, 선택적)
- Forward EPS, 목표가, EPS Revision
- 웹 스크래핑 필요 (공식 API 없음)
- 커버리지: 대형주 위주 (~300~500종목)
- 용도: Forward PER, Forward EPS 성장률, EPS Revision

### D. 네이버 금융 - 재무 요약 (보조)
- 재무제표 요약 (최근 4~5년), 일부 컨센서스
- 용도: DART 보완용

## 5. NAV 계산 상세 방법론

### Step 1: 지주사 식별
```
방법 1: 공정거래위원회 지주회사 현황 (가장 정확, 연 1회 업데이트)
방법 2: DART 기업개황에서 업종/사업내용에 '지주' 키워드
방법 3: PHASE0 마스터에서 섹터 + 종목명 '지주', '홀딩스' 포함 여부
→ 세 가지 결합하여 is_holdco 플래그 설정
```

### Step 2: 자회사 지분 구조 파악
```
DART API: dart.report(code, '지분현황') 또는 사업보고서 '종속회사 현황'
필요 정보:
  - 자회사명
  - 자회사 종목코드 (상장사인 경우)
  - 지분율 (%)
  - 상장/비상장 구분
```

### Step 3: NAV 산출
```python
def calculate_nav(holdco_code, subsidiaries, dart, pykrx_mcap):
    nav = 0
    
    # (1) 상장 자회사 지분가치 = 시가총액 × 지분율
    for sub in subsidiaries:
        if sub['listed']:
            sub_mcap = pykrx_mcap.get(sub['code'], 0)
            nav += sub_mcap * sub['ownership_pct'] / 100
    
    # (2) 비상장 자회사 지분가치 = 장부가(별도 재무제표의 투자주식)
    # DART 별도 재무제표에서 '관계기업투자주식' 항목
    nav += holdco_unlisted_investment_book_value
    
    # (3) 자체사업 가치 (별도 영업이익 × 멀티플)
    # 지주사 자체 사업의 영업이익 (별도 기준)
    own_biz_op = separate_operating_profit  # DART OFS
    own_biz_value = own_biz_op * sector_avg_ev_ebitda  # 또는 고정 8~10x
    nav += own_biz_value
    
    # (4) 순차입금 차감 (별도 기준)
    net_debt = total_borrowings - cash_and_equivalents  # DART OFS
    nav -= net_debt
    
    # NAV 디스카운트
    mcap = pykrx_mcap[holdco_code]
    nav_discount = 1 - (mcap / nav) if nav > 0 else None
    
    return nav, nav_discount
```

### Step 4: 지주사 스코어링 적용
- NAV 디스카운트율이 높을수록 → 밸류에이션 스코어 높음 (저평가)
- 나머지 카테고리(수익성, 성장성 등)는 **별도 재무제표** 기준 산출
- Forward 데이터는 동일하게 적용 (컨센서스는 대체로 연결 기준이므로 주의 필요)

## 6. 데이터 수집 전략 (3단계)

### Phase 2A: 기본 밸류에이션 (pykrx) — 즉시 가능
- 전종목 PER, PBR, EPS, BPS, DIV, 시가총액 일괄 수집
- ROE 근사값 = EPS / BPS × 100
- 기업 분류 (Type A/B/C) 설정
- **이 단계만으로도 기본 밸류에이션 스크리닝 가능**

### Phase 2B: DART 재무제표 + NAV — API 키 필요
- 연결 + 별도 재무제표 동시 수집
- 지주사 자회사 지분구조 파악
- NAV 계산 (상장 자회사 시총 × 지분율 + 비상장 장부가 - 순차입금)
- 수익성, 성장성, 건전성, 현금흐름 지표 산출
- API 호출량: ~1,271종목 × 2(연결+별도) × 최근 2년 = ~5,000건 (일 한도 내)

### Phase 2C: FnGuide 컨센서스 — 선택적
- Forward EPS, 목표가, EPS Revision 스크래핑
- 커버리지: 대형주 위주 (~300~500종목)
- Forward 데이터 없는 종목 → trailing만으로 스코어링 (가중치 재정규화)

## 7. 복합 스코어링

PHASE1과 동일한 카테고리 가중 Rank Percentile 프레임워크:

```
펀더멘탈 점수 = Σ (카테고리 가중치 × 카테고리 내 평균 rank_pct)

카테고리 가중치:
  밸류에이션 (30%): PER + PBR + Forward_PER + EV/EBITDA + 배당률
                    (지주사: NAV디스카운트 + 별도PER + 별도PBR + 배당률)
  수익성    (20%): ROE + 영업이익률 + 순이익률
  성장성    (25%): 매출YoY + 영업이익YoY + Fwd_EPS성장 + EPS_Revision
  재무건전성 (15%): 부채비율(역) + 유동비율 + 이자보상배율
  현금흐름  (10%): FCF_Yield + Accrual_Quality
```

### 스코어링 규칙
- 각 지표: 전종목 대비 rank percentile (0~1, 높을수록 우수)
- 방향이 '낮을수록' 지표: 1 - rank_pct로 반전
- 카테고리 내: 유효 지표의 평균 rank_pct
- 지표 데이터 없는 경우: 해당 카테고리 내 나머지 지표로 정규화
- 지주사(Type B): 밸류에이션 카테고리를 NAV 기반 지표로 대체
- 금융업(Type C): 영업이익률/부채비율 제외, 나머지 지표로 정규화

### 결과 출력
- 전종목 펀더멘탈 점수 + 순위 (Excel)
- 카테고리별 점수 히트맵 (PNG 차트)
- 섹터별 상위 종목 리스트
- 지주사 NAV 분석 시트 (별도 탭)

- DART 별도 재무제표: 일부 종목에서 별도 제출 안 하는 경우 있음 → 연결로 fallback
- 자회사 지분 데이터: DART '지분현황' 보고서 포맷이 비정형 → 파싱 난이도 높음

- FnGuide 스크래핑: 차단/구조변경 리스크 → graceful fallback
- 금융업종: 재무제표 구조 상이 → 별도 처리 (PER/PBR/ROE만 적용)
- 결산월 차이: 대부분 12월이나 일부 3월/6월 결산 → 최신 보고서 기준 조회
- Forward 데이터 미존재 종목: trailing만으로 스코어링, 가중치 재정규화
- NAV 계산의 비상장 자회사 가치: 장부가 기준 (보수적) → 추후 수익가치법 검토 가능
