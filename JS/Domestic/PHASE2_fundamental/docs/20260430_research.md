# PHASE2 Fundamental - 리서치 (v2)
Date: 2026-04-30

## 목표
Forward 컨센서스 기반 펀더멘탈 스크리닝 시스템
- PHASE1(모멘텀)과 완전 독립
- **Forward(향후 12개월)** 전망치 중심 평가
- 3대 스코어링: 밸류에이션(30%) + 성장성(40%) + 주주환원(30%)
- 재무건전성은 스코어링이 아닌 **필터**로 사용 (부실기업 사전 제거)
- 지주사는 NAV 조정으로 더블카운팅 해결

---

## 1. 설계 철학

### 기존 문제점
- 밸류에이션(PER, PBR)과 수익성(ROE, 영업이익률)이 본질적으로 중복
  → PER = 주가/EPS, ROE = EPS/BPS → 둘 다 수익성 기반 지표
  → 별도 카테고리로 나누면 수익성 좋은 종목에 이중 가점
- Trailing(과거) 지표 위주 → 과거 실적이 좋았던 종목이 아닌, 앞으로 좋아질 종목을 찾아야 함
- 재무건전성을 가중 스코어에 넣으면 안정적이지만 성장성 없는 종목이 상위에 오는 문제

### 새 설계 원칙
1. **Forward 중심**: 향후 12개월 컨센서스 기반 (과거 데이터는 보조)
2. **3 카테고리만**: 밸류에이션 / 성장성 / 주주환원 → 중복 없는 명확한 구분
3. **재무건전성 = 필터**: 부채비율 과다, 자본잠식, 이자보상배율 미달 종목은 사전 제거
4. **컨센서스 에버리징**: Forward 데이터는 복수 기관 추정치의 평균 사용

## 2. 데이터 소스

### A. FnGuide (comp.fnguide.com) — **핵심 소스**
Forward 컨센서스의 유일한 소스. 증권사 애널리스트 추정치 종합.
- Forward EPS (향후 12개월 주당순이익 컨센서스)
- Forward 매출액 / 영업이익 (향후 12개월)
- 목표가, EPS Revision (3개월 변화율)
- 커버리지: 애널리스트 리포트가 있는 종목만 (~300~500종목, 대형주 위주)
- 수집 방법: 웹 스크래핑 (공식 API 없음)
- **에버리징**: FnGuide가 이미 복수 증권사 추정치를 평균(컨센서스)으로 제공

### B. DART OpenAPI (OpenDartReader) — **재무제표 + 주주환원**
- 연결/별도 재무제표: 매출, 영업이익, 순이익, 자본, 부채
- 현금흐름표: 배당금 지급, 자기주식 취득/처분 금액
- 자회사 지분율: NAV 계산용
- API 키: 발급 완료 (일 10,000건)
- 용도: 주주환원 데이터, 재무건전성 필터, 성장률(trailing), NAV

### C. pykrx — **시총 + Trailing 기본**
- 전종목 PER, PBR, EPS, BPS, DIV, DPS, 시가총액
- KRX 로그인 필요 (.env에 설정 완료)
- 용도: 시가총액, Trailing PER/EPS (Forward 없는 종목 fallback), 배당수익률

## 3. 재무건전성 필터 (스코어링 전 사전 제거)

아래 조건 중 하나라도 해당하면 스크리닝 대상에서 **제외**:

| 필터 | 조건 | 소스 | 비고 |
|------|------|------|------|
| 자본잠식 | 자기자본 ≤ 0 | DART | 완전자본잠식 |
| 부채비율 과다 | 부채비율 > 400% | DART | 금융업 제외 |
| 이자보상배율 미달 | 영업이익/이자비용 < 1 | DART | 이자도 못 갚는 기업 |
| 관리종목/투자주의 | 거래정지, 관리종목 | KRX | 마스터에서 제외 |
| 적자 지속 | 3년 연속 영업적자 | DART | 턴어라운드 후보 제외 |

필터 통과 종목만 스코어링에 참여 → 결과적으로 기본 재무 안전판 역할

## 4. 스코어링 카테고리 (3개)

### 카테고리 A: Forward 밸류에이션 — 가중치 30%

**핵심 지표: Forward PER (향후 12개월 컨센서스 기준)**
= 현재 주가 / Forward EPS (컨센서스 평균)

| 지표 | 설명 | 소스 | 방향 |
|------|------|------|------|
| Forward PER | 현재가 / Forward EPS | FnGuide + pykrx | 낮을수록 저평가 |
| PEG Ratio | Forward PER / Forward EPS 성장률 | FnGuide | 낮을수록 성장대비 저평가 |

- Forward PER이 핵심. 수익성(ROE, 영업이익률)을 별도로 보지 않음
  → Forward EPS가 이미 수익성을 반영하고 있으므로
- PEG = PER/성장률 → 성장성 대비 밸류에이션 적정성 평가
- **Forward 데이터 없는 종목**: Trailing PER 사용 (가중치 50% 감소)
- **지주사(Type B)**: NAV 디스카운트율로 대체

### 카테고리 B: Forward 성장성 — 가중치 40%

모든 성장 지표를 **Forward(향후 12개월 전망)** 기준으로 산정

| 지표 | 설명 | 소스 | 방향 |
|------|------|------|------|
| Forward 매출 성장률 | 컨센서스 매출 / 직전 실적 매출 - 1 | FnGuide + DART | 높을수록 성장 |
| Forward 영업이익 성장률 | 컨센서스 영업이익 / 직전 실적 영업이익 - 1 | FnGuide + DART | 높을수록 성장 |
| Forward EPS 성장률 | 컨센서스 EPS / Trailing EPS - 1 | FnGuide + pykrx | 높을수록 성장 |
| EPS Revision | 최근 3개월 EPS 컨센서스 변화율 | FnGuide | 양수면 상향 |

- 성장성 비중이 가장 높음 (40%) → 미래 성장 가능성이 핵심 투자 기준
- EPS Revision은 시장 기대치 변화 방향을 반영 (모멘텀적 요소)
- **Forward 데이터 없는 종목**: Trailing YoY 성장률 사용 (가중치 50% 감소)

### 카테고리 C: 주주환원 — 가중치 30%

기업이 벌어들인 이익을 주주에게 얼마나 돌려주는지 평가

| 지표 | 설명 | 소스 | 방향 |
|------|------|------|------|
| 총 주주환원수익률 | (배당금 + 자사주매입) / 시가총액 | DART + pykrx | 높을수록 매력적 |
| 주주환원/영업이익 | (배당금 + 자사주매입) / 영업이익 | DART | 높을수록 적극적 (100% 이하) |
| 배당성향 | DPS / EPS (또는 배당총액/순이익) | pykrx or DART | 적정 수준 (20~60%) |
| 자사주매입비율 | 자사주매입금액 / 시가총액 | DART CF표 | 높을수록 매력적 |

**주주환원 데이터 산출 방법:**
```
DART 현금흐름표에서:
  배당금 지급액 = '배당금의지급' (재무활동CF 항목)
  자사주매입액 = '자기주식의취득' (재무활동CF 항목)
  자사주처분액 = '자기주식의처분' (재무활동CF 항목)

순 자사주매입 = 자기주식의취득 - 자기주식의처분
총 주주환원액 = 배당금 지급액 + 순 자사주매입

주주환원수익률 = 총 주주환원액 / 시가총액
주주환원/영업이익 = 총 주주환원액 / 영업이익
```

## 5. 지주사 NAV 조정 (유지)

기존 설계와 동일. 지주사(Type B)는 Forward PER 대신 NAV 디스카운트율로 밸류에이션 평가.

```
NAV = (상장 자회사 시총 × 지분율) + (비상장 자회사 장부가) + (자체사업 가치) - (순차입금)
NAV 디스카운트율 = 1 - (시가총액 / NAV)
```

성장성/주주환원은 별도 재무제표(OFS) 기준으로 동일하게 산출.

## 6. 스코어링 프레임워크

```
[필터 단계]
  재무건전성 필터 → 부실/위험 종목 제거

[스코어링 단계]
  펀더멘탈 점수 = Forward밸류에이션(30%) + Forward성장성(40%) + 주주환원(30%)

  카테고리 내: 각 지표의 rank percentile (0~1) → 평균
  카테고리 간: 가중 합산

[fallback 규칙]
  Forward 데이터 있는 종목: Forward 지표 100% 가중치
  Forward 데이터 없는 종목: Trailing 지표 사용, 해당 카테고리 가중치 50% 감소
  → 가중치 재정규화하여 합산
```

## 7. 데이터 수집 전략

### 수집 순서
1. pykrx: 전종목 PER/PBR/EPS/BPS/DIV/시총 (매일, 1분)
2. DART: 재무제표(연결+별도) + 현금흐름표 (분기 1회, ~5,000건)
3. FnGuide: Forward 컨센서스 스크래핑 (주 1회, ~500종목)

### Forward 데이터 커버리지 전략
- FnGuide 커버리지: 대형주 300~500종목 (컨센서스 있는 종목)
- 나머지 ~800종목: Trailing 데이터만 사용
- Trailing-only 종목도 스코어링 참여 (단, Forward 카테고리 가중치 50% 감소)
- 결과 시트에 Forward/Trailing 구분 표시

## 8. 기업 분류 체계 (유지)

### Type A: 일반기업 (~1,140종목)
- 연결 재무제표 기준 평가
- Forward 데이터 있으면 Forward 중심, 없으면 Trailing fallback

### Type B: 지주사 (~30종목)
- 밸류에이션: NAV 디스카운트율
- 성장성/주주환원: 별도(OFS) 재무제표 기준
- Forward 컨센서스 적용 시 주의 (대부분 연결 기준)

### Type C: 금융업 (~100종목)
- 매출/영업이익 구조가 다름 → 성장성은 순이익 기준
- 주주환원은 동일 적용
- 재무건전성 필터: 부채비율 기준 완화 (금융업 특성)

## 10. 리스크 및 고려사항

- **FnGuide 스크래핑**: 가장 큰 리스크. 차단/구조변경 시 Forward 데이터 수집 불가
  → Trailing fallback 설계로 대응. 네이버 금융 컨센서스를 보조 소스로 검토.
- **DART API 한도**: 일 10,000건 → 분기별 전체 수집 + 일일 증분
- **자사주매입 데이터**: DART 현금흐름표에서 추출. 항목명이 기업마다 다를 수 있음
  → '자기주식' 키워드로 유연하게 파싱
- **Forward 커버리지 편향**: 대형주만 컨센서스 존재 → 중소형주는 Trailing 스코어링
  → 결과에서 Forward/Trailing 구분 표시하여 해석 주의
- **지주사 NAV**: 비상장 자회사 가치 = 장부가 (보수적) → 실제보다 낮을 수 있음
- **금융업 주주환원**: 은행/보험사는 배당 비율이 높으므로 비금융 대비 유리할 수 있음
  → 섹터 내 상대 비교로 보정 검토
- **배당성향 비선형성**: 배당성향 100% 이상(적자인데 배당)은 비정상 → 캡 설정 필요
