- **최적화**: FinanceDatabase에서 1차 필터 후 대상만 호출
  - FinanceDatabase market_cap 정보가 있으면 $500M 이상만 yfinance 호출 (마진 확보)
  - 없으면 전종목 호출 (약 8,000~10,000건, ~1시간 소요)
- **rate limit 대응**: 0.3초 sleep, 500건 단위 배치, 진행률 표시
- **에러 처리**: 개별 종목 실패 시 스킵 & 로깅

## yfinance 호출 최적화

### 문제: 전종목(~10,000건) × 0.3초 = ~50분
### 해결책

1. **1차 필터**: FinanceDatabase에 market_cap 정보가 있으면 $500M 이상만 대상
   → 호출 대상 ~4,000~5,000건으로 축소

2. **배치 저장**: 500건마다 중간 결과 pkl 저장
   → 중단 시 이어서 수집 가능

3. **병렬 처리 (선택)**: ThreadPoolExecutor로 2~4개 동시 호출
   → 시간 50% 이상 단축 (rate limit 주의)

4. **캐시 활용**: 1회 수집 후 월간 갱신
   → 일상적으로는 캐시 사용

## 리스크 & 대응

| 리스크 | 영향 | 대응 |
|--------|------|------|
| FinanceDatabase 업데이트 지연 | 최근 IPO 누락 | yfinance에서 보충, 누락 종목 수동 추가 |
| yfinance 차단 | 시총/거래대금 수집 불가 | sleep 조절, IP 로테이션, FMP API 대안 |

| ADR 중복 | 본국+미국 이중 카운트 | is_adr 플래그로 구분, 필요 시 필터 |
| 거래대금 계산 오차 | avg_volume × price ≠ 실제 거래대금 | 근사치로 충분 (필터링 목적) |
