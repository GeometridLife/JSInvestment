# PHASE2 Fundamental - 구현 계획
Date: 2026-04-28

## 구현 범위

### 1단계: pykrx 기본 밸류에이션 + 기업 분류 (즉시 가능)

**scripts/data_collector.py**
- PHASE0 마스터 테이블에서 종목 리스트 로드
- pykrx `get_market_fundamental_by_ticker()` → PER, PBR, EPS, BPS, DIV
- pykrx `get_market_cap_by_ticker()` → 시가총액
- ROE 근사값 계산: EPS / BPS × 100
- 기업 분류 (Type A/B/C) 부여
  - Type C (금융업): WICS 섹터 기반
  - Type B (지주사): data/holdco_list.csv + 종목명 패턴 매칭
  - Type A (일반): 나머지
- 결과: cache/fundamental_cache.pkl

**scripts/fundamental_scoring.py (1단계 버전)**
- pykrx 데이터만으로 기본 스코어링
- 밸류에이션: PER, PBR, 배당수익률
- 수익성: ROE (근사값)
- 카테고리 가중 Rank Percentile
- 결과: results/YYYY/MM/YYYYMMDD_fundamental_ranking.xlsx

**data/holdco_list.csv (수동 작성)**
- 주요 지주사 목록 (종목코드, 종목명, 그룹명)
- 초기 20~30개 수동 매핑

### 2단계: DART 재무제표 + NAV (API 키 필요)

**scripts/data_collector.py (확장)**
- DART 연결 재무제표 수집 (finstate_all, fs_div='CFS')
- DART 별도 재무제표 수집 (finstate_all, fs_div='OFS')
- 영업이익, 순이익, 매출, 부채, 자본, 현금흐름 추출
- 최근 2~3년 데이터 (성장률 계산용)
- 증분 업데이트: 분기 보고서 발표 시기에 맞춰 갱신

**scripts/nav_calculator.py (신규)**
- data/holdco_subsidiaries.csv 로드 (지주사-자회사 매핑)
- 상장 자회사: pykrx 시총 × 지분율
- 비상장 자회사: DART 별도 재무제표의 관계기업투자주식
- 자체사업 가치: 별도 영업이익 × 적정 멀티플 (8~10x)
- 순차입금: 별도 기준 (차입금 - 현금)
- NAV, NAV 디스카운트율 산출
- 결과: cache/nav_cache.pkl

**scripts/fundamental_scoring.py (확장)**
- 수익성: ROE, 영업이익률, 순이익률 (DART 기준)
- 성장성: 매출 YoY, 영업이익 YoY
- 재무건전성: 부채비율, 유동비율, 이자보상배율
- 현금흐름: FCF Yield, Accrual Quality
- 지주사: NAV 디스카운트 적용
- 금융업: 별도 처리

**data/holdco_subsidiaries.csv (수동/반자동)**
- 지주사코드, 자회사코드, 자회사명, 지분율, 상장여부

### 3단계: FnGuide 컨센서스 (선택적)

**scripts/consensus_collector.py (신규)**
- FnGuide 웹 스크래핑
- Forward EPS, 목표가 수집
- Forward PER, EPS Revision 계산
- 결과: cache/consensus_cache.pkl

## 실행 순서

```
[1단계]
  data_collector.py (pykrx)
    → fundamental_scoring.py (기본 버전)
      → 결과 xlsx + 차트

[2단계]
  data_collector.py (pykrx + DART)
    → nav_calculator.py (지주사 NAV)
      → fundamental_scoring.py (전체 버전)
        → 결과 xlsx + 차트

[3단계]
  data_collector.py + consensus_collector.py
    → nav_calculator.py
      → fundamental_scoring.py (Forward 포함)
        → 결과 xlsx + 차트
```

STEPS = [

    ('Step 3: 펀더멘탈 스코어링', 'fundamental_scoring.py'),
]

## 코드 설계 원칙

1. PHASE1과 동일한 패턴: SCRIPT_DIR → BASE_DIR → 하위 폴더 경로
2. 캐시 파일: cache/ 폴더에 pkl, 날짜 없는 고정 파일명 (매일 덮어쓰기)
3. 결과: results/YYYY/MM/ 폴더에 날짜 prefix 포함
4. 로그: logs/ 폴더에 실행 이력
5. 증분 업데이트: pykrx는 매일, DART는 분기별 (보고서 발표 주기)
6. 에러 처리: graceful fallback (데이터 없으면 해당 지표 스킵)
