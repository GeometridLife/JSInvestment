## 1. 섹터 분류 체계 비교

### A. GICS (Global Industry Classification Standard) ⭐ 권장
- 개발: MSCI + S&P Dow Jones (1999년~)
- 구조: 4단계 계층
  - Sector (11개) → Industry Group (25개) → Industry (74개) → Sub-Industry (163개)
- 최신 업데이트: 2023년 3월 (Internet & Direct Marketing Retail 해체, Data Center REITs 신설 등)
- 장점: 글로벌 투자업계 표준, ETF/펀드 벤치마크와 일치, 투자자 관점 분류
- 단점: 공식 데이터는 유료 (MSCI/S&P 라이선스)

**GICS 11개 Sector:**
1. Energy (에너지)
2. Materials (소재)
3. Industrials (산업재)
4. Consumer Discretionary (경기소비재)
5. Consumer Staples (필수소비재)
6. Health Care (헬스케어)
7. Financials (금융)
8. Information Technology (정보기술)
9. Communication Services (커뮤니케이션서비스)
10. Utilities (유틸리티)
11. Real Estate (부동산)

### B. SIC (Standard Industrial Classification)
- 개발: SEC/미국 정부
- 구조: 4자리 코드 (10개 Division → ~83 Major Group → ~416 Industry)
- 장점: SEC EDGAR에서 무료로 제공, 전종목 커버
- 단점: 1987년 기준으로 오래됨, IT/바이오 등 현대 산업 분류가 부실
- 용도: SEC 공시 분류용으로만 사용, 투자 분석에는 부적합

### C. NAICS (North American Industry Classification System)
- SIC 후속, 미국/캐나다/멕시코 공동
- 투자 분석보다 경제 통계용

### 결론
→ **GICS 기준 채택** (Domestic의 WICS가 GICS의 한국 변형인 것과 일맥상통)
→ GICS 공식 데이터는 유료이므로, GICS를 근사하는 무료 소스를 활용

## 2. 데이터 소스 비교

### A. FinanceDatabase (financedatabase PyPI) ⭐⭐⭐
- GitHub: github.com/JerBouma/FinanceDatabase
- 내용: 300,000+ 종목 (글로벌), Equities/ETFs/Funds 등
- 분류: GICS 근사 기반 sector/industry 제공
- 필터: country, sector, industry, exchange별 필터링
- 중복상장: `only_primary_listing` 파라미터로 1차 상장만 추출 가능
- 가격: 완전 무료 (오픈소스)
- 제한: 정적 데이터베이스 (업데이트 주기가 느릴 수 있음), 시총/거래량 정보 없음
- 용도: **종목 리스트 + 섹터 분류의 베이스**

### B. yfinance ⭐⭐⭐
- GitHub: github.com/ranaroussi/yfinance
- 내용: Yahoo Finance 데이터 (개별 종목 info에 sector, industry 포함)
- 시총: `info['marketCap']` 제공
- 거래량: `info['averageVolume']`, `info['averageDailyVolume10Day']` 제공
- 가격: 무료 (비공식 Yahoo Finance API)
- 제한: 종목 수가 많으면 rate limit / IP 차단 위험, 종목별 개별 호출 필요
- 용도: **시총/거래량 필터링, 섹터 정보 크로스체크**

### C. Financial Modeling Prep (FMP) API ⭐⭐
- URL: financialmodelingprep.com
- 내용: Stock Screener API (sector, industry, marketCap, volume 필터)
- 무료: 250 requests/day, US 종목만
- 유료: $14/month~ (더 많은 호출)
- 장점: 한 번의 API 호출로 조건별 종목 리스트 반환 (screener endpoint)
- 단점: 무료 한도 250건/일로 전종목 스크리닝에 제한
- 용도: **보조 소스 / 크로스밸리데이션**

### D. SEC EDGAR (company_tickers.json + SIC)
- URL: sec.gov/files/company_tickers.json
- 내용: 전체 SEC 등록 기업 리스트 (ticker, CIK, company name)
- SIC 코드: EDGAR full-text search로 SIC 코드 매핑 가능
- 가격: 완전 무료 (정부 공개 데이터)
- 제한: SIC 분류 자체가 오래됨, GICS 매핑 필요
- 용도: **전종목 리스트 확보의 보완 소스**

### E. NASDAQ Screener CSV
- URL: nasdaq.com/market-activity/stocks/screener (CSV 다운로드)
- 내용: NYSE, NASDAQ, AMEX 전종목 (Name, Sector, Industry, Market Cap, Country)
- 분류: 자체 섹터 분류 (GICS와 유사하나 완전 동일하지 않음)
- 가격: 무료 (수동 다운로드 or 스크래핑)
- 제한: 스크래핑 차단 가능성
- 용도: **전종목 리스트 + 섹터의 또 다른 소스**

## 3. 데이터 수집 전략

### 권장 조합: FinanceDatabase + yfinance (2단계)

**Step 1: FinanceDatabase → 종목 리스트 + 섹터 분류 (벌크)**
```python
import financedatabase as fd

equities = fd.Equities()
us_stocks = equities.select(country='United States')
# → ticker, name, sector, industry_group, industry, exchange, market_cap 등
# → ~8,000~10,000종목 (미국 전체)
```
- 장점: 한 번에 전종목 + 섹터 정보 확보
- API 호출 부담 없음 (로컬 데이터베이스)

**Step 2: yfinance → 시총, 거래량, 섹터 크로스체크 (종목별)**
```python
import yfinance as yf

ticker = yf.Ticker('AAPL')
info = ticker.info
# sector, industry, marketCap, averageVolume, exchange, quoteType 등
```
- 필터링 대상 종목에 대해서만 호출 (전종목 X)
- rate limit 관리: 0.5~1초 sleep, 배치 처리

### 대안: FMP Screener API (1단계로 가능)
```
GET /api/v3/stock-screener?marketCapMoreThan=1000000000&exchange=NYSE,NASDAQ&apikey=...
```
- 장점: 한 번의 호출로 필터링된 종목 리스트 반환
- 단점: 무료 250건/일, 전체 종목 스캔 시 여러 날 필요

## 4. 필터링 기준

### Domestic 기준 (참고)
- 시총: ≥ 1,000억 원 (~$70M)
- 일평균 거래대금: ≥ 10억 원 (~$700K)

### Global (미국) 기준 제안
| 필터 | 기준 | 사유 |
|------|------|------|
| 시가총액 | ≥ $1B (1조원 상당) | 미국 시장 규모 고려, Micro-cap 제외 |
| 일평균 거래량 | ≥ 500K shares/day | 유동성 확보 |
| 종목 유형 | Common Stock만 | ETF, Warrant, ADR, Preferred 등 제외 |
| 거래소 | NYSE, NASDAQ, AMEX | OTC 제외 |
| 국가 | US 본사 (선택적) | ADR/외국기업 포함 여부 결정 필요 |

→ 예상 통과 종목: ~1,500~2,500개 (Domestic 1,271개와 유사한 스케일)

### 시총 기준 대안 검토
| 기준 | 통과 종목 수 (추정) | 특징 |
|------|---------------------|------|
| ≥ $10B (Large Cap) | ~500~700 | S&P 500 수준, 대형주만 |
| ≥ $2B (Mid Cap+) | ~1,500~2,000 | 중형주 이상 |
| ≥ $1B | ~2,000~2,500 | 소형주 일부 포함 |
| ≥ $300M (Small Cap+) | ~3,500~4,000 | 소형주 포함 |

→ $1B~$2B 사이가 Domestic과 유사한 밀도

## 5. 종목 유형 필터링

미국 시장은 한국 대비 종목 유형이 다양하여 필터링이 중요:

| 유형 | 포함 여부 | 사유 |
|------|-----------|------|
| Common Stock | ✅ | 핵심 분석 대상 |
| ADR (American Depositary Receipt) | ✅ | TSMC, 알리바바 등 대형 글로벌 기업 포함 |
| Preferred Stock | ❌ | 채권 성격, 모멘텀 분석 부적합 |
| ETF | ❌ | 개별 종목 아님 |
| SPAC | ❌ | Domestic과 동일 이유 |
| Warrant | ❌ | 파생상품 |
| Unit | ❌ | 복합 증권 |
| REIT | ✅ | GICS에 Real Estate 섹터 존재, 포함 |

→ ADR 포함 확정: TSMC, ASML 등 핵심 글로벌 기업 커버리지 확보

## 6. GICS vs Domestic WICS 매핑

| GICS Sector | 대응 WICS tier1 |
|-------------|-----------------|
| Energy | 에너지/화학 (일부) |
| Materials | 철강/소재, 에너지/화학 (일부) |
| Industrials | 기계/산업재, 방산/항공우주 |
| Consumer Discretionary | 소비재/유통, 자동차/부품 |
| Consumer Staples | 소비재/유통 (일부) |
| Health Care | 바이오/제약/헬스케어 |
| Financials | 금융/보험 |
| Information Technology | 반도체, IT서비스/소프트웨어, IT하드웨어/전자부품 |
| Communication Services | 통신, 미디어/엔터테인먼트, 인터넷/플랫폼 |
| Utilities | 전력인프라/에너지장비 |
| Real Estate | 건설/부동산 |

→ GICS 11개가 WICS 18개보다 상위 레벨. Industry Group (25개)이 WICS tier1과 유사한 granularity.

## 8. 리스크 및 대응

| 리스크 | 대응 |
|--------|------|
| financedatabase 데이터 오래됨 | yfinance로 크로스체크, 최신 IPO 종목은 yfinance에서 보충 |
| yfinance rate limit / 차단 | sleep 간격, 배치 처리, 필터링 대상만 호출 |
| GICS 분류와 불일치 | financedatabase의 sector/industry를 기준으로 하되, 명백한 오류는 수동 보정 |
| ADR/외국기업 혼재 | quoteType, country 필터로 제어 |
| 우선주/ETF 혼입 | quoteType == 'EQUITY' 필터 |
| 섹터 미분류 종목 | 'Unknown' 또는 빈 값 → 별도 시트에 리스트업 |

## 9. 확정된 의사결정

1. **시총 기준**: ≥ $1B — Domestic(1,271개)과 유사한 ~2,000~2,500종목 스케일
2. **ADR 포함**: ✅ 포함 — TSMC, 알리바바 등 대형 글로벌 기업 커버리지 확보
3. **분류 granularity**: GICS Sector(11개) 메인 + Industry Group(25개) 참조
4. **거래량 기준**: 거래대금 기준 ($10M/day) — 가격대가 다른 종목 간 공정한 비교
5. **데이터 갱신 주기**: 월간 — 섹터 분류 자체는 자주 변하지 않음
