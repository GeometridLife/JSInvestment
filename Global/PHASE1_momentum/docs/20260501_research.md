## 1. 데이터 소스

### A. yfinance `yf.download()` ⭐ 확정
- **필수**: yfinance **1.3.0+** (0.2.x는 Yahoo API 변경으로 완전 실패)
- **핵심**: `yf.download(tickers, period, group_by='ticker')` — 여러 종목 한 번에 다운로드
- Ticker.info(개별 호출)와 달리 download()는 벌크 호출이라 rate limit이 훨씬 관대
- 10년치 일간 OHLCV 제공
- 무료, API 키 불필요

**⚠️ yfinance 1.3.0 주의사항:**
- 컬럼이 항상 MultiIndex `(metric, ticker)` 형태로 반환됨
- 단일 종목이라도 `data['Close']['AAPL']` 또는 `data[('Close','AAPL')]` 로 접근
- `auto_adjust=True` 시 Adj Close 컬럼 없이 Close가 수정주가

**테스트 결과 (2026-05-01):**
- 80종목 × 1년: 5.5초, 80/80 성공, rate limit 없음
- 10종목 × 10년: 1.2초, 2,514영업일
- 연속 3배치 + 2초 sleep: 전부 성공
- 데이터 품질: 이상치 0건, Volume=0 없음

**벌크 다운로드 전략:**
```python
import yfinance as yf

# 80종목 단위 배치 (권장 chunk size)
tickers = ['AAPL', 'MSFT', ...]  # 2,280개
chunk_size = 80

for i in range(0, len(tickers), chunk_size):
    chunk = tickers[i:i+chunk_size]
    data = yf.download(
        tickers=chunk,
        period='10y',
        interval='1d',
        group_by='ticker',
        auto_adjust=True,
        threads=True,
    )
    # 종목별 분리 저장
    time.sleep(2)  # 배치 간 쿨다운
```

**예상 소요:**
- 2,280종목 ÷ 80 = 29 배치
- 배치당 ~10초(다운로드) + 2초(쿨다운) = ~6분
- Domestic(FDR 기준 30~60분) 대비 크게 단축

**Rate limit 대응:**
- 80종목 단위 배치 + 2초 sleep
- 429 에러 시 30초 대기 후 재시도 (최대 3회)
- 중간 저장(pickle)으로 중단 시 재개 가능

## 2. 모멘텀 지표 설계 (Domestic과 동일)

### 2-1. DD (Drawdown) — 5개 서브 지표

| 서브 지표 | 정의 | 용도 |
|-----------|------|------|
| Forward_MDD (무조건부) | 매 시점 t에서 향후 20영업일간 최대 낙폭 분포 | 종목 간 하방 리스크 비교 |
| Forward_MDD (조건부) | 현재 DD 수준별 향후 20영업일간 최대 낙폭 분포 | 현재 시점 진입 타이밍 판단 |
| Recovery Time | DD 이벤트 후 전고점 복귀까지 영업일 수 (중앙값) | 회복 속도 평가 |
| Calmar Ratio | 연환산 수익률 / MDD | 수익 대비 리스크 종합 랭킹 |
| DD Velocity | DD 크기 / DD 도달 기간(일) | 급락 vs 완만하락 구분 |

조건부 Forward_MDD 구간: 0~5%, 5~10%, 10~20%, 20%+

### 2-2. MACD — 교차점 기반

- MACD Line: EMA(12) - EMA(26)
- Signal Line: EMA(9) of MACD Line
- Histogram: MACD - Signal
- **진입**: Histogram 골든크로스 (음→양 전환)
- **청산**: Histogram 데드크로스 (양→음 전환)
- **평가**: 누적 수익률 (로그수익률, 왕복 수수료 차감)

### 2-3. RSI — 과매도 반등 전략

- RSI 기간: 14일 (Wilder 표준)
- **진입**: RSI가 30 하회 후 30 상향 돌파
- **청산**: RSI가 70 상회 후 70 하향 돌파
- **평가**: 누적 수익률 (비용 반영)

### 2-4. 52주 신고가 — 브레이크아웃 전략

- **진입**: 52주(250영업일) 신고가 갱신 당일
- **청산**: 신고가 대비 -5% 하락 시 (트레일링 스톱)
- **평가**: 누적 수익률 (비용 반영)
- 동일 종목 연속 진입 방지: 최소 5영업일 간격

- 한국 증권사 온라인 수수료 기준

## 6. 랭킹 및 선별 기준

### 6-1. 종목 선별 (섹터 무관, 전체 랭킹)
- **지표별 Top 20**: 각 지표의 누적 수익률 기준 상위 20종목 (4개 지표 × 1장 = 4장 차트)

### 6-2. 섹터 선별 (GICS 섹터별 랭킹)
- **섹터별 Top 3**: 11개 GICS 섹터 × 4개 지표별 Top 3 (4개 지표 × 1장 = 4장 차트)

### 6-3. 복합 모멘텀
- 4개 지표 중 2개 이상에서 Top 50에 드는 종목 전체 (1장 차트)

### 차트 출력 총 9장
| 차트 | 내용 | 종목 수 |
|------|------|---------|
| 종목 Top 20 × 4 | 지표별 전체 랭킹 상위 20 | ~20 |
| 섹터 Top 3 × 4 | 섹터별 대표 종목 | ~33 (11×3) |
| 복합 모멘텀 1 | 2개+ 지표 Top 50 교집합 | 가변 |
