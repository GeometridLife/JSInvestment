## 2. 설계 철학 (Domestic 동일)

### 핵심 원칙
1. **Forward 중심**: 향후 12개월 컨센서스 기반. 과거 데이터는 fallback.
2. **밸류에이션 × 성장성 곱셈 스코어**: 둘 다 좋아야 점수가 높음. 한쪽만 좋은 종목은 자동 패널티.
3. **주주환원 = 부가지표**: 별도 카테고리·별도 랭킹. 메인 스코어 미반영.
4. **재무건전성 = 사전 필터**: 부실기업 제거. 스코어링 대상에서 빠짐.
5. **그룹별 랭킹**: 시총 기준 대형/중형/소형 분리 후 그룹 내 비교 (대형주가 중소형주를 압도하는 현상 방지).
6. **Forward fallback 페널티**: Forward 없는 종목은 Trailing 사용하되 점수 × 0.50.

### 왜 곱셈인가
- 가중합(밸류 30% + 성장 40% + 주주 30%)은 한쪽이 매우 좋으면 다른 쪽이 나빠도 상위에 오를 수 있음
- 곱셈은 둘 다 일정 수준 이상이어야 점수 형성 → "싸면서 잘 크는" 종목만 상위
- Domestic에서 검증된 패턴

## 3. 재무건전성 필터 (사전 제거) — 다중 시그널 + 섹터별 임계값

### 설계 원칙
1. **자본잠식만 단독 제외**: 회복 불가 시그널이라 단독 위반으로도 제외
2. **나머지 4개는 다중 시그널 (2개 이상 위반 시 제외)**: 단일 지표 라벨 매핑 누락·일시 회계 이슈로 인한 false negative(좋은 회사 잘못 제외) 방지
3. **NaN-safe**: 라벨 매칭 못 한 항목은 'unknown' = 위반 아님 (통과 처리). `data_completeness` 컬럼에 NaN 개수 표시
4. **섹터별 임계값**: 산업 특성 반영 (금융 면제, REIT 완화 등)

### 필터 정의

| # | 필터 | 조건 | 종류 | 비고 |
|---|------|------|------|------|
| 1 | **자본잠식** | StockholdersEquity ≤ 0 | **단독 제외** | 회복 불가 |
| 2 | 고부채 | (LongTermDebt + LongTermDebtCurrent) / StockholdersEquity > **섹터 임계값** | 다중 시그널 | 금융 면제 |
| 3 | 이자보상미달 | OperatingIncome / abs(InterestExpense) < 1 | 다중 시그널 | 금융 면제 |
| 4 | 3년 연속 영업적자 | OperatingIncome < 0 for 3y | 다중 시그널 | NaN 면제 |
| 5 | 3년 연속 영업CF 음수 | OperatingCashFlow < 0 for 3y | 다중 시그널 | **신규** — 회계 적자보다 더 신뢰성 높은 시그널 |
| 6 | 거래정지·시총 누락 | marketCap NaN 또는 0 | **단독 제외** | 마스터에서 제외 |

**제외 조건**: 1번 또는 6번 단독 위반 / 또는 2~5번 중 **2개 이상 위반**

### 섹터별 D/E 임계값 (필터 2)

| GICS 섹터 | D/E cutoff | TIE 적용 (필터 3) | 비고 |
|---|---|---|---|
| **Financials** | **면제** | **면제** | 부채가 영업 자산 (예금/보험금) |
| **Real Estate** (REIT) | **8.0** | 1.0 | 자산을 부채로 매입하는 구조 |
| **Utilities** | **5.0** | 1.0 | 자본집약적 |
| 그 외 9섹터 | **4.0** | 1.0 | 일반 |

### 신규 필터 — 영업CF 3년 연속 음수
회계 적자(OperatingIncome)는 일시적 사유(R&D, 감가상각, 일회성 비용)로도 발생하지만, **영업 현금흐름 적자가 3년 지속되면 진짜 부실 시그널**. Buffett 류 quality 평가의 표준 비기준이며, 회계조작 회피에도 강함.

|---|---|---|---|---|---|---|
| 정상 회사 | ✓ | ✓ | ✓ | ✓ | ✓ | **통과** |
| TIE만 0.8 | ✓ | ✓ | ✗ | ✓ | ✓ | 통과 (1개) |
| 자본잠식 | ✗ | - | - | - | - | **단독 제외** |

## 4. 스코어링 카테고리

### 카테고리 A: 밸류에이션
지표가 낮을수록(=싼 종목) 좋음. rank_pct(ascending=True).

| 지표 | 정의 | 소스 | 비고 |
|------|------|------|------|
| Forward PER | 현재가 / forwardEps | yfinance + 계산 | 핵심 |
| PEG Ratio | Forward PER / Forward EPS 성장률(%) | yfinance | 성장 대비 밸류 |
| Trailing PER | 현재가 / Trailing EPS | yfinance + EDGAR | Forward 없을 때 fallback |

- Forward PER이 핵심. Trailing만 쓰는 종목은 최종 스코어에 0.50 페널티
- 미국시장 지주사 NAV는 제거 (Berkshire 등 극소수 — 별도 처리 불필요)

### 카테고리 B: 성장성
지표가 높을수록 좋음. rank_pct(ascending=False).

| 지표 | 정의 | 소스 |
|------|------|------|
| Forward 매출 성장률 | revenueGrowth | yfinance |
| Forward 이익 성장률 | earningsGrowth (≈ EPS YoY) | yfinance |
| Forward EPS 성장률 | (forwardEps / trailingEps) - 1 | yfinance + EDGAR |
| EPS Revision | yfinance에 직접 필드 없음 → forwardEps 30일/90일 변화 추적 (별도 캐싱 필요) | yfinance + 자체 캐시 |

- Trailing 폴백: EDGAR에서 매출/영업이익/순이익 YoY 계산 (2년치 비교)
- Forward 4지표 중 1개라도 잡히면 forward로 처리, 전부 없으면 Trailing fallback (페널티)

### 카테고리 C: 주주환원 (부가지표, 메인 미반영)

| 지표 | 정의 | 소스 |
|------|------|------|
| Total Shareholder Yield | (DividendsPaid + StockRepurchase) / marketCap | EDGAR + yfinance |
| Return / OperatingIncome | (DividendsPaid + StockRepurchase) / OperatingIncome | EDGAR |
| Payout Ratio | DividendsPaid / NetIncome | EDGAR |
| Buyback Yield | StockRepurchase / marketCap | EDGAR |

- 카테고리 점수는 산출하지만 **fundamental_score(메인)에는 반영 안 함**
- 별도 `shareholder_rank` 컬럼으로 그룹별 랭킹 제공
- 결과 시트에서 메인 랭킹과 주주환원 랭킹 둘 다 보여줌

## 5. 스코어링 프레임워크

```
[Phase 1: 필터]
  재무건전성 필터 → 부실/위험 종목 제거 (1,841 → ~1,500 추정)

[Phase 2: 카테고리 점수]
  cat_valuation = mean(fwd_per_rank, peg_rank)            # Forward 우선
                  또는 mean(trailing_per_rank)            # Trailing fallback
  cat_growth    = mean(fwd_rev/op/eps_rank, eps_rev_rank) # Forward 우선
                  또는 mean(trailing_rev/op/eps_rank)     # Trailing fallback
  cat_shareholder = mean(sh_yield, return_to_op, payout, buyback)  # 부가지표

[Phase 3: 메인 스코어]
  raw_score = cat_valuation × cat_growth   # 곱셈
  
  penalty = 1.0
  if not val_is_forward:    penalty *= 0.50
  if not growth_is_forward: penalty *= 0.50
  
  fundamental_score = raw_score × penalty

[Phase 4: 그룹별 랭킹]
  size_group = 'Large'  if marketCap >= $10B
             = 'Mid'    if $2B <= marketCap < $10B
             = 'Small'  if marketCap < $2B

  size_rank        = 그룹 내 fundamental_score 순위
  shareholder_rank = 그룹 내 cat_shareholder 순위 (별도)

[Phase 5: 출력]
  - Excel: 그룹별 랭킹 시트, Top50, 섹터별Top10, 필터제외종목
  - 차트: 메인 Top30 (그룹별), 밸류vs성장 산점도, 섹터 평균, 주주환원 분포
```

### 시총 분류 기준 (미국시장)
| 그룹 | 시총 |
|------|------|
| Large | ≥ $10B |
| Mid | $2B ~ $10B |
| Small | < $2B |

S&P/Russell 표준에 가까움. Domestic은 1조원/3000억원 기준이었지만 미국시장 사이즈 분포에 맞춰 재정의.

### Forward fallback 페널티 = 0.50
- val_is_forward = forwardPE 잡힌 종목
- growth_is_forward = forward 성장 4지표 중 1개라도 잡힌 종목
- 둘 다 trailing이면 페널티 = 0.50 × 0.50 = 0.25

| Forward fallback 페널티 0.50 | ✓ | ✓ (동일) |
| 주주환원 부가지표 분리 | ✓ | ✓ (동일) |

## 8. 랭킹 및 출력

### 8-1. 그룹별 메인 랭킹
- 대형/중형/소형 각 그룹 내 fundamental_score 순위
- Top 30 시각화 × 3 (그룹별) = 3장 차트

### 8-2. 섹터별 랭킹
- GICS 11섹터 × 그룹 (Large 내 섹터별 Top 5 등)
- 또는 그룹 무관 섹터별 Top 10

### 8-3. 주주환원 별도 랭킹
- 그룹별 cat_shareholder 순위
- Top 30 시각화 = 1장

## 9. 리스크 및 대응

| 리스크 | 대응 |
|--------|------|
| EDGAR 라벨 다양성 (회사·섹터별) | candidate fallback 패턴, 본구현 시 50종목으로 매핑 검증 |
| 금융주 (JPM 등) 손익 구조 다름 | sector-aware 매핑 모듈, 필터 기준 별도 설정 |
| Forward 컨센서스 yfinance 의존 | 출처 불투명 — Bloomberg/Refinitiv 같은 유료 대체 시 즉시 교체 가능하도록 모듈 분리 |
| 2,280종목 EDGAR 풀스캔 시간 (~7시간) | 첫 실행만 느림. 이후 분기 증분 (~30종목, 5분). 캐시 적극 활용 |
| yfinance rate limit | 0.5초 sleep, 429 시 30초 대기 + 3회 재시도, 100종목마다 중간 저장 |
| EDGAR vs yfinance 값 불일치 | yfinance.cashflow를 검증 채널로 사용. 차이 5% 이상 시 로그 기록 + 수동 검토 |
| StockRepurchase None vs 0 구분 | EDGAR 라벨 자체가 없으면 None(공시 안 함), 라벨 있고 값 0이면 0 |
| ADR 펀더멘탈 처리 | 본 PHASE에서 제외 (439종목). 모멘텀(PHASE1)에는 그대로 포함 |
| Forward fallback 비중 과다 | 결과에서 forward/trailing 종목 비율 표시, 50% 미만이면 경고 |
